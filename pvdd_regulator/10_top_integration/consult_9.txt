2. NODE DEFINITIONS
================================================================================

  Node        Description                      Key impedance
  ────        ───────────                      ──────────────
  d2          Stage 1 output (drain)           Rd2 = ro1||ro2 ~ few MΩ
                                               Cd2 = Cparasitic ~ 0.5-1pF
  vout_gate   Stage 2 output (EA output)       Rout2 ~ few hundred kΩ
  gate        Pass device gate                 Rgate=200Ω to vout_gate

3. TRANSFER FUNCTION ELEMENTS
================================================================================

Stage 1 (Folded-cascode EA, differential pair):
  ─ Transconductance: gm1 (set by tail current, ~10-50 µA/V typ)
  ─ Output: current into node d2
  ─ d2 impedance: Rd2 = ro_pmos || ro_nmos (very high, ~MΩ)
  ─ d2 parasitic cap: Cd2 ~ 0.5-1 pF

Miller compensation network (d2 to vout_gate):
  ─ Cc = 40 pF (series with Rc = 5 kΩ)
  ─ Creates: dominant pole splitting, LHP zero, RHP zero

Stage 2 (NFET common-source/cascode):
  ─ Transconductance: gm2 ~ 300 µA/V (estimated from p2 placement)
  ─ Output: current into node vout_gate
  ─ Rout2: output resistance of cascode (high, ~MΩ)

Gate drive network:
  ─ Rgate = 200 Ω (series resistance, ea_out to gate)
  ─ Csnub = 100 pF (gate to BVDD, AC ground)
  ─ Cgs_pass ~ 5 pF (gate-source of pass PMOS)

Pass device (PMOS source follower in common-source config):
  ─ gm_pass: depends on Iload. At 10mA ~ few mA/V
  ─ Drives current into pvdd node
  ─ Note: Vsg = Vbvdd - Vgate, so iout = -gm_pass * Vgate
    (inverting: gate going low → more current → pvdd rises)

Output node:
  ─ Rload: depends on load current (∞ at no-load, ~500Ω at 10mA)
  ─ Cout = 1 µF (external) + 200 pF (on-chip) ≈ 1 µF total

Feedback:
  ─ beta = R_BOT / (R_TOP + R_BOT) = 118k / (364k + 118k) = 0.245
  ─ Cfb = 2 pF across R_TOP (FIX-11, adds a feedback zero/pole pair)


4. POLES AND ZEROS
================================================================================

─────────────────────────────────────────────────────────────────────────────
  Name    Type        Frequency       Formula
─────────────────────────────────────────────────────────────────────────────
  p1      Pole        ~300 Hz         1/(2π * Rout_total * Cout)
          (dominant)                  Rout_total ≈ (rds_pass || Rload)
                                      With Miller splitting, pushed very low

  p2      Pole        ~1.2 MHz        gm2 / (2π * Cc)
          (non-dom)                   = 300µ / (2π * 40p) ≈ 1.19 MHz

  p3      Pole        ~7.6 MHz        1/(2π * Rgate * (Cgs + Csnub))
          (gate)                      = 1/(2π * 200 * 105p) = 7.58 MHz

  p_fb    Pole        ~220 kHz        1/(2π * (R_TOP||R_BOT) * Cfb)
          (feedback)                  = 1/(2π * 89k * 2p) ≈ 894 kHz
                                      BUT: in loop, appears at ~220 kHz
                                      due to interaction with divider

  z1      Zero (LHP)  ~796 kHz        1/(2π * Rc * Cc)
          (Miller Rc)                 = 1/(2π * 5k * 40p) = 796 kHz

  z2      Zero (RHP)  ~1.2 MHz        gm2 / (2π * Cc)
          (Miller)                    = 300µ / (2π * 40p) ≈ 1.19 MHz
                                      (cancels p2 approximately — depends
                                       on exact gm2 vs Rc*gm2 ratio)

  z_fb    Zero (LHP)  higher          From Cfb bypassing R_TOP at HF
─────────────────────────────────────────────────────────────────────────────

Note: p2 and z2 are both at ~gm2/(2πCc). The exact locations depend on
whether gm2*Rc > 1 or < 1:
  ─ If gm2*Rc = 300µ * 5k = 1.5 > 1: z1 (from Rc) is below p2, good.
    The RHP zero z2 = gm2/Cc is at same frequency as p2.
  ─ In practice, with Rc nulling, the RHP zero moves to:
    z2_eff = 1/(Cc * (1/gm2 - Rc))
    If Rc > 1/gm2: RHP zero becomes LHP zero! (both zeros are LHP)
    1/gm2 = 1/300µ = 3.33 kΩ.  Rc = 5 kΩ > 3.33 kΩ.
    → z2 is ALSO LHP at: 1/(2π*Cc*(Rc - 1/gm2)) = 1/(2π*40p*1.67k)
                        = 2.39 MHz (LHP)
    This is GOOD for stability.


5. OPEN-LOOP GAIN
================================================================================

DC loop gain T(0):
  T(0) = gm1 * Rd2 * gm2 * Rout2 * gm_pass * Rload * beta

  Measured: 70.8 dB = 3467 V/V

With single dominant pole at p1 = 300 Hz, gain crosses 0 dB at:
  f_UGB = T(0) * f_p1 = 3467 * 300 = 1.04 MHz  (if single-pole rolloff)

But measured UGB ≈ 142 kHz (from Bode sim), suggesting additional
gain reduction from parasitic poles and/or lower effective DC gain at
the operating point simulated.

Alternatively, the gain slope steepens before 0 dB crossing due to p2,
pulling UGB back. Let's use the measured value: UGB ≈ 142 kHz.

7. HUNTING THE MISSING POLES
================================================================================

The measured ~50° phase margin at ~142 kHz means total phase lag ≈ -130°.
Our 5-singularity model gives only -84.3°. We're missing ~46° of phase lag.

Possible sources of the missing phase:

(a) Parasitic pole at node d2:
    ─ p_d2 = 1/(2π * Rd2 * Cd2)
    ─ Without Miller: if Rd2 ~ 5 MΩ, Cd2 ~ 1 pF → f = 32 kHz
    ─ WITH Miller splitting, this pole gets pushed to p2 = gm2/Cc
    ─ But if Cd2 is larger (layout parasitics), or if Miller splitting
      is imperfect, a residual mid-frequency pole could remain.
    ─ If effective p_d2 ~ 500 kHz: adds -arctan(142k/500k) = -15.8°

(b) Feedback pole from Cfb = 2 pF:
    ─ Cfb across R_TOP creates a pole in the feedback path
    ─ f_pfb = 1/(2π * R_TOP * Cfb) = 1/(2π * 364k * 2p) = 219 kHz
    ─ At 142 kHz: -arctan(142k/219k) = -arctan(0.649) = -33.0°
    ─ THIS IS HUGE. A 33° phase loss from Cfb alone!

(c) Pole inside the folded-cascode (current mirror pole):
    ─ PMOS mirror in EA stage 1 has a pole from Cgs of mirror devices
    ─ Typically at a few MHz, adds a few degrees

(d) Distributed RC of poly resistors:
    ─ R_TOP = 364 kΩ in xhigh_po: distributed RC with substrate
    ─ Could add a few degrees of excess phase

Let's check if (b) alone explains the discrepancy:

  With p_fb at 219 kHz added:
  ──────────────────────────────────────────────────────────────────────
  Total phase = -84.3° + (-33.0°) = -117.3°
  Phase margin = 180° - 117.3° = 62.7°
  ──────────────────────────────────────────────────────────────────────

  Still 12.7° more optimistic than measured 50°. The remaining ~13° likely
  comes from (a) parasitic pole at d2 and (c) mirror poles.

  If we add a lumped "parasitic" pole at ~800 kHz:
  -arctan(142k/800k) = -10.1° → total = -127.4° → PM = 52.6°  ✓ MATCHES!

10. THE REAL CULPRIT: Cfb = 2 pF
================================================================================

The feedback capacitor Cfb = 2 pF across R_TOP = 364 kΩ creates a pole at:

  f_pfb = 1/(2π * R_TOP * Cfb) = 1/(2π * 364k * 2p) = 219 kHz

This is only 1.54× above UGB (142 kHz). At UGB it contributes:

  φ_pfb = -arctan(142k/219k) = -arctan(0.649) = -33.0°

  ┌─────────────────────────────────────────────────────────────┐
  │  Cfb PHASE PENALTY AT UGB: -33°                             │
  │                                                             │
  │  This single capacitor eats 33° of phase margin!            │
  │  It was added in FIX-11 to filter HF noise on vfb.         │
  │  But it's destroying loop stability.                        │
  └─────────────────────────────────────────────────────────────┘

What Cfb does to the feedback:
  ─ At DC: beta = R_BOT/(R_TOP+R_BOT) = 0.245
  ─ At f >> f_pfb: Cfb shorts R_TOP, so beta → 1.0
  ─ The transition happens around 219 kHz — right near UGB
  ─ This creates a POLE in the loop gain (beta increases → more feedback
    → but with phase lag from the RC)

Actually, more precisely: Cfb across R_TOP creates a zero in beta(s)
at f_z = 1/(2πR_TOP*Cfb) = 219 kHz and a pole at f_p = 1/(2π*(R_TOP||R_BOT)*Cfb)
= 1/(2π*89k*2p) = 894 kHz.

In the LOOP GAIN T(s) = A(s)*beta(s):
  ─ The zero in beta appears as a ZERO in loop gain at 219 kHz
  ─ The pole in beta appears as a POLE in loop gain at 894 kHz
  ─ Net effect near UGB: the zero at 219 kHz BOOSTS gain (pushing UGB higher)
    but the associated phase lead/lag interaction reduces phase margin.

Wait — let me reconsider. If Cfb creates a zero in beta(s):
  beta(s) = (R_BOT * (1 + s*R_TOP*Cfb)) / (R_TOP + R_BOT + s*R_TOP*R_BOT*Cfb)

  Zero at: 1/(2π*R_TOP*Cfb) = 219 kHz → ADDS phase (LHP zero in beta)
  Pole at: 1/(2π*(R_TOP||R_BOT)*Cfb) = 894 kHz → SUBTRACTS phase

  At UGB = 142 kHz:
    Phase from beta_zero (219 kHz):  +arctan(142k/219k) = +33.0°
    Phase from beta_pole (894 kHz):  -arctan(142k/894k) = -9.0°
    Net from Cfb: +33.0° - 9.0° = +24.0° of phase LEAD

Hmm, that's phase LEAD, not lag. But Cfb also increases |beta| near UGB,
which pushes UGB to a higher frequency where other poles contribute more lag.

REVISED ANALYSIS — Cfb effect on UGB:
  ─ |beta| at DC = 0.245
  ─ |beta| at 142 kHz: boosted by ~10-20% due to Cfb zero
  ─ |beta| at 1 MHz: approaching 1.0 (Cfb shorts R_TOP)
  ─ Higher beta → higher loop gain → higher UGB → more phase from p2, p_par

  If UGB increases from 142 kHz to (say) 200 kHz due to beta boost:
    p2 at 200 kHz: -arctan(200k/1.2M) = -9.5° (was -6.8°)
    p_par at 200 kHz: -arctan(200k/800k) = -14.0° (was -10.1°)
    Additional lag: ~6.6°

This is complex. The net effect of Cfb depends on the interaction.

POSSIBLE EXPLANATIONS FOR THE DISCREPANCY:

(1) UGB is actually higher than 142 kHz.
    ─ The Cfb zero at 219 kHz boosts beta, increasing loop gain.
    ─ If actual UGB ≈ 500 kHz - 1 MHz, phase margin drops significantly.
    ─ At UGB = 800 kHz:
      p1: -90°, p2: -arctan(0.667)=-33.7°, p3: -6.0°,
      p_par: -45°, p_fb_pole: -arctan(0.895)=-41.8°
      z1: +arctan(1.005)=+45.2°, z2: +arctan(0.335)=+18.5°,
      z_fb: +arctan(3.65)=+74.7°
      Total = -90 -33.7 -6.0 -45 -41.8 +45.2 +18.5 +74.7 = -78.1°
      PM = 180 - 78.1 = 101.9°  (still too high)

(2) There are additional HF poles we haven't accounted for.
    ─ Pass device: Cgd creates a Miller pole seen at the drain
    ─ Multiple poles inside the folded-cascode
    ─ Distributed effects in long poly resistors

(3) The phase margin was measured as CLOSED-LOOP from step response,
    not directly from Bode. Ringing with ~50° PM was inferred from
    transient overshoot, which can be misleading if there are
    complex-conjugate poles.

(4) The DC gain is load-dependent, and at the operating point in
    the Bode sim, gm_pass may be very high, pushing UGB well above
    142 kHz into the region where p2 and p_par really bite.


12. KEY INSIGHT: WHAT ACTUALLY LIMITS PHASE MARGIN
================================================================================

From the pole-zero map, the phase margin is primarily set by the
SPACING between UGB and the first non-dominant pole cluster (p_fb_pole,
p_parasitic, p2) all bunched at 800 kHz - 1.2 MHz.

If UGB is truly at 142 kHz (0.14 MHz), these poles at ~1 MHz are 7×
above UGB, contributing only ~25° total lag. That should give PM > 60°.

But if the actual UGB (accounting for Cfb beta-boost and gm_pass
variation with load) is closer to 400-600 kHz, then:
  ─ Poles at 800k-1.2M are only 1.3-3× above UGB
  ─ Each pole contributes 20-40° of lag
  ─ Phase margin collapses to 40-55°

14. RECOMMENDATIONS FOR PHASE MARGIN IMPROVEMENT
================================================================================

Based on this analysis, the main stability levers are:

(A) REDUCE UGB by increasing Cc or reducing gm1:
    ─ Cc: 40pF → 80pF would halve UGB, gaining ~15° PM
    ─ But: slower transient response, already 9ms settling

(B) PUSH NON-DOMINANT POLES HIGHER:
    ─ Reduce R_TOP||R_BOT to push p_fb_pole higher
      (but changes beta → changes PVDD)
    ─ Reduce Cfb: 2pF → 0.5pF moves p_fb_pole to 3.6 MHz
      and z_fb to 874 kHz. Less beta-boost near UGB.
      Cost: less HF filtering on vfb.

(C) MOVE z1 CLOSER TO UGB for more phase boost at crossover:
    ─ Currently z1 = 796 kHz. If UGB ≈ 500 kHz, z1 is only 1.6× above.
    ─ Move z1 lower: Rc*Cc product must increase.
    ─ Rc: 5kΩ → 20kΩ with Cc=40pF → z1 = 199 kHz
    ─ Or Rc: 5kΩ → 10kΩ → z1 = 398 kHz
    ─ Risk: if z1 goes below UGB, it boosts gain and pushes UGB up

(D) REMOVE or REDUCE Cfb:
    ─ Cfb was added in FIX-11 for HF noise filtering
    ─ If Cfb is removed: no p_fb_pole, no beta boost, UGB drops,
      PM improves significantly (~20-30°)
    ─ If HF noise is a concern, use a smaller Cfb (0.2-0.5pF)
      to push the pole above 1 MHz

RECOMMENDED NEXT STEP:
  Simulate Bode plot with Cfb = 0 (remove it) and compare PM.
  If PM improves to >65°, then use Cfb = 0.2-0.5pF as compromise.
