## 1. Architecture

```
 8 Feature Inputs          4× MAC Units (8in×4bit)          WTA
 ┌────────────┐     ┌─────────────────────────┐     ┌──────────────┐
 │ BPF1 env   ├────►│ MAC_0: Normal weights   ├────►│              │
 │ BPF2 env   ├────►│ MAC_1: Imbalance wts    ├────►│  3 StrongARM │
 │ BPF3 env   ├────►│ MAC_2: Bearing weights  ├────►│  comparators │──► 2-bit class
 │ BPF4 env   ├────►│ MAC_3: Looseness wts    ├────►│  (binary     │    (00/01/10/11)
 │ BPF5 env   ├────►│                         │     │   tree WTA)  │
 │ RMS        ├────►│ 32 binary-weighted caps  │     └──────────────┘
 │ Crest      ├────►│ per MAC (MIM + parasitic)│
 │ Kurt       ├────►│ 161 transistors/MAC      │       3-Phase Clock
 └────────────┘     └─────────────────────────┘     ┌──────────────┐
                            │                        │ NAND-based   │
                     ┌──────▼──────┐                │ non-overlap  │
                     │ 3-phase clk │◄───────────────│ clk_in→φs,  │
                     │ φs, φe, φr  │                │ φe, φr       │
                     └─────────────┘                └──────────────┘
```

### Charge-Sharing MAC Principle (SPICE-verified)

1. **Sample (φs):** TG charges cap top plate to input voltage. Bottom plate grounded. Q = C × Vin.
2. **Evaluate (φe):** Sample TGs open. Eval TGs connect top plates to bitline. Charge redistribution:
   **Vbl = Σ(C_wi × V_fi) / (Σ C_all + Cpar)** — verified to 0.4% accuracy in ngspice.
3. **Reset (φr):** Bitline + all top plates discharged to 0V.

### 3.3 Priority 4: Monte Carlo — Two Independent Methods

#### Method 1: Python-based MC (200 runs, analytical mismatch model)

![Monte Carlo Python](plots/p4_monte_carlo.png)

Pelgrom cap mismatch (A_C = 0.45 %·μm) + kT/C noise + comparator offset (σ_Vth = 3.54 mV):

#### Method 2: SPICE-driven MC (50 ngspice runs, per-instance cap variation)

![Monte Carlo SPICE](plots/mc_spice_mim.png)

Each of 50 runs generates a **unique ngspice netlist** with per-instance MIM cap W/L variation derived from Pelgrom mismatch, then runs a full transient simulation. This is transistor-level MC — not behavioral.

| Class | Mean Vbl [mV] | σ [mV] | Notes |
|---|---|---|---|
| Normal | 539.5 | 0.048 | Dominant winner |
| Imbalance | 520.2 | 0.044 | |
| Bearing | 511.1 | 0.041 | |
| Looseness | 449.7 | 0.043 | |

- **SPICE MC accuracy: 100.0%** (50/50 correct) — PASS (>85%)
- **ENOB: 4.0 bits** per class — PASS (≥4)

The extremely low σ (~0.04–0.05 mV) is because the 6.36 pF total bitline capacitance provides excellent averaging of individual cap mismatches. The class separation (19.3 mV minimum) is ~400× larger than the MAC output variation.

**MC approach:** Native ngspice MC (`mc_mm_switch=1`) is set in the library but the extracted pm3 BSIM4 models don't reference the slope parameters internally. The Python-driven approach (`run_mc_spice.py`) achieves physically-equivalent MC by varying each MIM cap's W/L dimensions per-instance according to the Pelgrom model. This generates real ngspice simulations — not behavioral math.

## 5. Honest Assessment

### What Works

1. **Full-scale 8×4-bit MAC verified in ngspice.** 161 transistors, 32 binary-weighted caps with bottom-plate parasitics, all simulated with real SKY130 BSIM4 models. Not behavioral — real transistor-level.

2. **Linearity is excellent.** 0.08 LSB error after offset correction. The charge domain Q=CV is inherently linear.

3. **Process robustness is extraordinary.** ±0.11% across 5 corners. Capacitor ratio accuracy is the key advantage.

4. **WTA works with 19 mV margin.** StrongARM correctly resolves the winning class. 99.5% accuracy under Monte Carlo mismatch.

5. **Power is essentially zero.** The duty cycle at 10 Hz is so low (500ns active per 100ms period = 5 ppm) that average power is dominated by leakage.

### What Doesn't Work / Limitations

1. **Monte Carlo uses Python-driven ngspice.** Each MC run generates a unique SPICE netlist with per-instance W/L variation and runs real ngspice. The `mc_mm_switch=1` is set in the library but the extracted pm3 models don't reference slope parameters internally. The Python-driven approach is physically equivalent but not identical to the full combined PDK's native MC engine.

2. **MIM cap model is a subcircuit, not the full PDK model.** We use `sky130_fd_pr__cap_mim_m3_1.spice` with correct area (2 fF/μm²) + fringe (0.38 fF/μm) + bottom-plate parasitic (0.1 fF/μm²) from the xschem symbol. The voltage coefficient (~50 ppm/V) is not modeled — for 0–1.8V range this introduces <0.01% error.

3. **Comparator offset is 0 mV in schematic-level SPICE.** Real silicon will have 3-10 mV offset. The MC analysis accounts for this probabilistically but a transistor-level MC sim would be more accurate.

4. **No layout parasitics.** The 80 fF bitline parasitic is an estimate. Real routing parasitics depend on metal stack and routing length.

5. **Classification test uses one feature vector.** Priority 9 (100+ test vectors) is needed for full accuracy characterization with CWRU data.

6. **Clock generator needs non-overlap verification.** The waveforms show correct phasing but exact non-overlap timing needs measurement at all corners.

7. **Weight loading interface (Priority 6) not yet designed.** The 128-bit shift register for SPI weight loading is needed for a complete design.

### Failed Approaches

1. **First MAC architecture (top-plate to bitline, bottom floating):** Charge injection caused 80% error. Bottom plates must be grounded. Discovered during SPICE debugging of initial 4×2-bit prototype.

2. **Series switch topology (sample TG + weight NMOS):** The NMOS-only weight switch couldn't pass voltages above VDD-Vth. Had to switch to pre-computed AND(phi_s, weight) enable signals with full TG.
